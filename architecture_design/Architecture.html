OleaBot – Simple Pipeline Architecture

Simple flow: Satellite + Weather + Farm Data → Feature Builder → AI Message → Telegram/WhatsApp updates

1
Satellite (Sentinel-2 / GEE)
NDVI / NDMI
Input
  • Farm polygon (GeoJSON)
  • Date window (ex: last 30 days)
Output
  • ndvi_current, ndvi_delta
  • cloud_pct, date_used
↓
2
Weather (Open-Meteo)
ET0 + Rain
Input
  • lat / lon (farm centroid)
  • 7-day forecast
Output
  • et0_daily[7], rain_daily[7]
  • et0_week, rain_week
↓
3
Web Page (Farmers Onboarding)
Profile + Polygon
Collect
  • Soil type, slope (optional)
  • Tree age, tree count, spacing
  • Farm polygon (draw on map)
Store in DB
  • FarmerProfile
  • FarmPlot (GeoJSON)
  • Telegram chat_id (for updates)
↓
4
Feature Builder (+ FAO-56 baseline)
Numbers Ready
Build features
  • Combine Satellite + Weather + Farm Data
  • Compute FAO-56 irrigation (litres/tree)
  • Create one JSON “feature vector”
Output (example)
  • litres_per_tree, total_m3
  • ndvi_current, ndvi_delta
  • et0_week, rain_eff, phase
↓
5
AI Module (Cloud API + Prompt)
Explain + Tips
AI does
  • Write message (clear + friendly)
  • Explain satellite change (NDVI trend)
  • Give tips (best hours, stress warning)
AI does NOT
  • Replace FAO-56 math
  • Decide random litres without rules
↓
6
Bot (Telegram / WhatsApp)
Updates Only
Send
  • Weekly irrigation update
  • Alert if stress detected
Log
  • AlertRecord (sent_at, litres/tree)
  • ndvi_current, ndvi_delta (audit)

MVP endpoints (simple):
POST /register (profile + polygon)  |  POST /calculate?farm_id= (run pipeline)  |  POST /dispatch/weekly (send updates)
