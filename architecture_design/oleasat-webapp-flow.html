🌿 Web App — Flow + Technical Spec

From Farmer's Field
to Smart Dashboard

The farmer uploads or draws their olive zone. The system fetches satellite data, computes NDVI + water stress indices, and shows a live farm dashboard with irrigation status.

Web Only (No Mobile) FastAPI Backend Next.js Frontend Draw Polygon + Upload File NDVI / NDWI / Stress Dashboard Sentinel-2 via GEE PostGIS Database
01 — Complete User Journey
Every Step the Farmer Takes

From landing on the site to reading today's irrigation recommendation — here is every screen and action in order.

01
Page: / (Landing)
Farmer arrives — creates account
A clean landing page explains the value: "Know exactly when and how much to irrigate, using satellite data — no sensors needed." The farmer clicks Sign Up, enters name + phone + email. Backend creates the user, returns a JWT token, farmer is redirected to the dashboard.
✉️
Register FormName, phone, email, password. Language selector (FR / AR / EN).
🔑
JWT AuthToken stored in httpOnly cookie. No localStorage.
➡️
RedirectOn success → /dashboard (empty state, prompt to add first parcel).
↓ auth success → /dashboard
02
Page: /dashboard (Empty State)
First visit — no parcels yet
Dashboard is empty. A prominent call-to-action button says "Add Your First Field". The farmer clicks it and a modal opens with two tabs: Draw on Map, or Upload File.
🗺️
Empty StateIllustration + "Add Your First Field" button. Clean, not cluttered.
➕
Modal OpensTwo-tab interface: Draw Mode or Upload Mode.
↓ farmer clicks "Add Field"
03
Modal Tab A: Draw on Map
Option A — Draw the zone on satellite map
A Leaflet map loads centered on Morocco. Satellite tiles (Mapbox or ESRI). The farmer zooms to their land, clicks the Draw button, and clicks around the edges of the field. Each click adds a vertex. Double-click closes the polygon. The boundary glows green to confirm.
🛰️
Satellite TilesESRI World Imagery or Mapbox Satellite as base layer so the farmer can see their actual land.
✏️
Leaflet.drawPolygon drawing tool. Shows area (ha) in real time as polygon grows.
📐
Live StatsArea in hectares, estimated tree count at default density (100 trees/ha).
03
Modal Tab B: Upload File
Option B — Upload GeoJSON, KML, or Shapefile
The farmer (or their agronomist) may already have a digital boundary file. They drag and drop it into a dropzone. The system parses the file, extracts the polygon geometry, and renders it on the Leaflet map immediately. If the file has multiple features, the farmer picks which one.
📂
Supported FormatsGeoJSON (.geojson), KML (.kml), Shapefile (.zip of .shp + .dbf + .prj).
🗺️
Instant PreviewPolygon is rendered on map immediately after upload. Green fill with border.
⚙️
Backend ParseFastAPI uses Shapely + Fiona to validate and standardize geometry to WGS84.
↓ polygon confirmed → farmer fills in field details
04
Modal Step 2: Field Metadata
Name the field + add details
After drawing or uploading, a small form appears in the same modal: Field name, crop type (defaults to Olive), soil type (Clay / Sandy / Loam), tree density, irrigation method. Then "Save Field". Backend stores the polygon in PostGIS and triggers an async job.
🌿
Metadata FormName, crop type, soil type, tree density/ha, irrigation method (drip / flood / none).
💾
POST /parcelsGeoJSON + metadata saved. Backend returns parcel_id.
⚡
Job TriggeredCelery worker starts extracting Sentinel-2 data for this polygon (async).
↓ save → polygon on dashboard, data loading...
05
Page: /dashboard/:parcelId
The Farm Dashboard — Live Status
This is the core screen. The farmer sees their field boundary on the map, and 4 KPI cards at the top: NDVI today, Water Stress Index, Days Since Rain, and Today's Action. Below: an NDVI time series chart (last 6 months), a weather chart, and the irrigation recommendation card. Data refreshes every time a new Sentinel-2 image is available (every 5 days).
🟢
KPI CardsNDVI, NDWI stress index, days without rain, current recommendation action.
📈
Index ChartsNDVI + NDWI time series (interactive, hover to see exact date values).
🌧️
Weather ChartDaily rainfall + ET₀ bars for the last 30 days.
💧
Irrigation CardIRRIGATE / SKIP / URGENT + liters per tree + confidence score.
↓ farmer has multiple fields → uses sidebar to switch between them
06
Ongoing: Multi-Field Management
Manage all fields from one sidebar
The left sidebar lists all the farmer's parcels with a colored status dot (green = OK, amber = moderate stress, red = urgent). The farmer can click any parcel to switch the dashboard view. They can also add new fields anytime via the "+" button, rename or delete existing ones.
📋
Parcel SidebarList of all fields, status dot, area, last update timestamp.
🔄
Auto-RefreshData updates automatically every 5 days when new Sentinel-2 pass is processed.

04 — Backend Technical Flow
What Happens Behind the Scene

From the moment the farmer saves their field, here is every technical step the backend executes — automatically.

→
Step 1
Receive Polygon
FastAPI · PostGIS
POST /parcels receives GeoJSON
Validate geometry with Shapely
Reproject to EPSG:4326
Store in parcels table (PostGIS)
Compute area_ha via ST_Area
→
Step 2
Trigger Async Job
Celery · Redis
API pushes task to Redis queue
Worker picks up extract_parcel task
Date range: today − 180 days
Parcel marked status = "processing"
→
Step 3
Satellite Extraction
GEE · Sentinel-2
Filter S2 collection by polygon + date
Cloud mask (SCL band ≤ 20%)
Compute NDVI, NDWI, NDSI per image
Reduce to mean per polygon
Return time series as DataFrame
→
Step 4
Weather + Features
ERA5-Land · Pandas
Fetch ERA5: rain, temp, ET₀
Align dates to satellite time series
Compute lag features (5/10/30 days)
Rolling stats: rain_14d, et0_7d
Store in satellite + meteo tables
Step 5
Generate Recommendation
XGBoost → TFT
Send features to ML inference service
Get stress_score + water_need_mm
Map to SKIP / IRRIGATE / URGENT
Convert mm → liters/tree using density
Store in recommendations table
Mark parcel status = "ready"
All API Routes Used by the Web App
FastAPI — Base URL: /api/v1
POST/auth/registerCreate new farmer account, returns JWT
POST/auth/loginReturns JWT token in httpOnly cookie
GET/auth/meCurrent user profile
GET/parcelsList all parcels for current user
POST/parcelsCreate parcel (body: GeoJSON + metadata), triggers extraction job
GET/parcels/{id}Single parcel detail + current status (processing / ready / error)
GET/parcels/{id}/indicesNDVI + NDWI + NDSI time series (query: start_date, end_date)
GET/parcels/{id}/meteoDaily rainfall + temperature + ET₀ history
GET/parcels/{id}/recommendToday's recommendation: action, liters/tree, confidence, key factors
POST/parcels/{id}/refreshManually trigger new data extraction (if farmer wants latest update)
POST/upload/geojsonParse uploaded GeoJSON file → returns polygon coordinates for preview
POST/upload/shapefileParse .zip shapefile → returns GeoJSON polygon for preview
POST/upload/kmlParse KML → returns GeoJSON polygon for preview
05 — Full Technology Stack
Every Tool, Every Layer

Chosen for simplicity, speed to build, and production readiness. No overengineering.

Layer Technology Why This Choice Used For
Frontend Next.js 14 React File-based routing, SSR for SEO, huge ecosystem All 5 pages, charts, map
Map Leaflet.js + leaflet-draw + react-leaflet Free, polygon drawing built-in, GeoJSON native, satellite tiles Field boundary display + drawing tool
Charts Recharts React React-native, responsive, interactive tooltips, line + bar NDVI/NDWI time series, weather bar chart
File Parse (front) toGeoJSON.js + JSZip Parses KML and Shapefile in browser before upload Upload tab: instant polygon preview
Styling Tailwind CSS Utility-first, fast, consistent design tokens All UI components
API Framework FastAPI Python 3.11 Async, auto OpenAPI docs, best Python AI ecosystem integration All REST endpoints, auth, file handling
Auth python-jose + passlib JWT Lightweight, standard, no extra service needed Login, register, protected routes
Geospatial Shapely + Fiona + pyproj + GDAL Parse/validate any polygon format, reproject, compute area Upload parsing, area calculation
Satellite Google Earth Engine API geemap Free for research, Sentinel-2 built-in, cloud masking NDVI/NDWI extraction per polygon
Weather ERA5-Land (CDS API) + Open-Meteo ERA5 for history, Open-Meteo for free 7-day forecast Rain, ET₀, temperature history + forecast
Task Queue Celery + Redis Run GEE extraction async without blocking API (can take 30-60s) Data extraction jobs, scheduled daily refresh
Database PostgreSQL 16 + PostGIS SQLAlchemy Native geometry storage, spatial queries (ST_Area, ST_Intersects) Users, parcels, time series, recommendations
ML (Phase 1) XGBoost + SHAP scikit-learn Works with 500 samples, fast inference, explainable feature importance Stress score → irrigation decision
ML (Phase 2) pytorch-forecasting (TFT) MLflow 60-day attention window, multi-input (sat + meteo + static) 7-day forecast, higher accuracy
SMS Alerts Africa's Talking API Morocco coverage, cheaper than Twilio, Arabic SMS support Push irrigation alerts to farmer's phone
DevOps Docker + docker-compose GitHub Actions Reproducible local dev, easy deployment to any VPS Local dev, CI/CD, production deploy
Hosting MVP Railway + Vercel Railway for FastAPI + Postgres + Redis; Vercel for Next.js frontend Fast deployment, free tiers for MVP
